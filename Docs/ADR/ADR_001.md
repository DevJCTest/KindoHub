# ADR 001: Elección de C# y SQL Server como tecnologías principales


## Contexto y problema

Para el desarrollo de KindoHub es necesario elegir un lenguaje de programación para la API y un sistema de gestión de bases de datos. Esta decisión es fundamental ya que afectará directamente a la velocidad de desarrollo, la calidad del código, el rendimiento del sistema y la capacidad de mantener y escalar la aplicación a largo plazo.

## Opciones consideradas

- Opción A: C# con SQL Server - Utilizar C# como lenguaje de programación para la API y SQL Server como sistema de gestión de bases de datos.
- Opción B: Java con PostgreSQL - Utilizar Java como lenguaje de programación y PostgreSQL como base de datos relacional.
- Opción C: Python con MySQL - Utilizar Python para el desarrollo de la API y MySQL como sistema de gestión de bases de datos.

## Decisión

Se ha decidido implementar la opción A, utilizando C# como lenguaje de programación para la API y SQL Server como sistema de gestión de bases de datos.


## Justificación

- Se ha elegido la opción A porque C# y SQL Server son las tecnologías con las que se tiene mayor experiencia y conocimiento, lo que permite un desarrollo más rápido y eficiente.
- La familiaridad con el ecosistema .NET y sus herramientas permite implementar soluciones robustas con mejores prácticas de manera más natural.
- El conocimiento de SQL Server permite aprovechar sus características avanzadas como tablas temporales, rowversion, índices optimizados, y otras funcionalidades nativas que mejoran el rendimiento y la funcionalidad de la aplicación.
- La opción B, aunque es una combinación sólida y muy utilizada en la industria, requeriría una curva de aprendizaje adicional tanto en Java como en las particularidades de PostgreSQL.
- La opción C, si bien Python es un lenguaje muy popular y MySQL es ampliamente utilizado, la menor experiencia con estas tecnologías podría resultar en tiempos de desarrollo más largos y menor aprovechamiento de características avanzadas.

## Consecuencias

### Positivas

- Desarrollo más rápido y eficiente al trabajar con tecnologías conocidas, lo que reduce el tiempo de implementación y permite entregar valor más rápidamente.
- Menor probabilidad de errores al utilizar herramientas y patrones ya conocidos y probados en proyectos anteriores.
- Mejor capacidad para resolver problemas complejos y optimizar el rendimiento del sistema aprovechando el conocimiento profundo de las tecnologías.
- Acceso directo al ecosistema .NET con sus múltiples bibliotecas, herramientas y recursos de la comunidad.
- Posibilidad de aprovechar características avanzadas de SQL Server como tablas temporales para auditoría, rowversion para concurrencia optimista, entre otras.

### Negativas / Riesgos

- Posible dependencia del ecosistema Microsoft, aunque con .NET Core/.NET las aplicaciones son multiplataforma.
- Los costos de licenciamiento de SQL Server pueden ser elevados en producción (aunque existen ediciones gratuitas como Express y Developer para ciertos escenarios).
- Menor flexibilidad para migrar a otras tecnologías en el futuro si fuera necesario, debido a la familiaridad específica con estas herramientas.
- Posible sesgo hacia soluciones específicas de SQL Server que podrían no ser fácilmente portables a otros sistemas de bases de datos.

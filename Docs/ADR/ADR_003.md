# ADR 006: Enmascaramiento automático de IBANs con columna computada PERSISTED

**Estado:** Aceptado  
**Fecha:** 2026-02-14  
**Decisores:** [Tu Nombre] (Backend Developer), Equipo Seguridad  
**Relacionado con:** Tabla IBAN_Datos, RGPD Artículo 32, ADR 004 (RowVersion)

## Contexto y Problema

Nuestra aplicación gestiona IBANs bancarios sensibles (`ES9121000832990200507333`) para familias. Necesitamos cumplir **RGPD Artículo 32** (Seguridad del tratamiento) minimizando exposición de datos sensibles:

- **Usuarios normales** : Solo ven `ES************507333`
- **Admins autorizados** : Ven IBAN completo


## Opciones consideradas

| Opción | Descripción | Pros | Contras |
|--------|-------------|------|---------|
| **A: Columna computada PERSISTED** | `IBAN_Enmascarado AS LEFT(IBAN,2)+'*'+RIGHT(IBAN,4)` | Automático, indexable, zero código app, logs seguros | Solo SQL Server |
| **B: View con CASE WHEN rol** | `CASE WHEN @IsAdmin THEN IBAN ELSE MASK END` | Portable, granular por rol | No indexable, código por query |

## Decisión

**Implementar columna computada `PERSISTED`** en `IBAN_Datos`:

```sql
[IBAN_Enmascarado] AS (
    LEFT([IBAN_Completo],2) + 
    REPLICATE('*', LEN([IBAN_Completo])-6) + 
    RIGHT([IBAN_Completo],4)
) PERSISTED
